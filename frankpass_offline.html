How to Use FrankPass: The Ultimate Guide

Welcome to FrankPass. This tool is designed to generate unbreakable passwords while keeping your mental load near zero.

Core Principle: This tool is 100% Stateless and Deterministic. It doesn't save your data anywhere. No databases. It's pure math running between your browser and a secure stateless API. If you enter the exact same details, you will always get the exact same password.

Even if a hacker reads this guide, they cannot steal your passwords because the security relies on what's in your head, not what's stored in a system.

🔑 Understanding the Inputs

  1. Platform / Website (Required): This changes your password. Always use a consistent name (e.g., facebook).
  2. Username / Email (Optional): This changes your password. Use exact spelling (e.g. master_manikant is different from mastermanikant). If you don't use it, leave it blank.
  3. Secret Personal Key (Required): This is your master phrase. Use words and numbers that are easy for you to remember but hard for others to guess (e.g., ihavenoidea1988 or iamking20).
    Note: Capital letters and spaces do not matter in the Secret Key. (e.g., I am Good = iamgood = IAMGOOD).
  4. Variant Counter (Default 1): Keep your Secret Key the same. Change this number to generate a new password for the same site (e.g., if you need to change your Facebook password, just increase Variant from 1 to 2).
  5. Profile / Length: These only change the format of the password, but changing them will give a different password sequence.

🧠 The "Zero Mental Load" Strategy

The Golden Rule: Keep your Secret Key, Variant, and Length exactly the same for 99% of websites. Only change the Platform name! This way, you only need to remember your Secret Key.

⚠️ Important Do's and Don'ts

  • If a site requires fewer characters: Lower the length slider and regenerate.
  • If a site doesn't allow symbols (!@#): Change the Profile to "Alphanumeric".
  • Keep your Clipboard clean by copying random text after logging in.
  • Never write down the final password or your Secret Key. Write down the recipe instead: "Facebook - Username X - Variant 2".

Frequently Asked Questions

Q: What is the Security Architecture of FrankPass?

A: FrankPass uses 100% client-side cryptography. Advanced hashing algorithms (PBKDF2 and HMAC-SHA512) run locally in your browser to generate deterministic output. Because there are no databases or cloud syncing involved, there is nothing for hackers to breach. It is mathematically unbreakable and completely serverless.

Q: What happens if I forget my Secret Key?

A: Because FrankPass is zero-knowledge and stateless, there is no "Forgot Password" or recovery feature. Your Secret Key is the mathematical foundation of your passwords. We recommend using a memorable passphrase or your deeply ingrained personal logic.

Q: Will I get the same password if I switch devices?

A: Absolutely. FrankPass is deterministic. As long as you input the exact same 'Platform Name', 'Secret Key', and 'Variant Counter', the math will output the exact same generated password on any smartphone, tablet, or desktop, entirely offline.
